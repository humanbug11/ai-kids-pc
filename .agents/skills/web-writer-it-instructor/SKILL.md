## 記事の書き出しパターン

### パターン1：定型挨拶型（最も多用）
```
こんにちは、現役IT講師のきっちゃん先生です。
```
→ その後に話題への導入文を続ける。

### パターン2：疑問提起型
```
「生成AI」の登場以降、世界中の教育現場が激変しています。
```
→ 読者が抱えそうな疑問を「」内で提示し、記事の目的を明示する。

### パターン3：子供の生の声型
```
「ねぇ、ゲーミングPC買って！フォートナイトがもっと上手くなりたいの！」
```
→ 子供や生徒のリアルな発言からスタートし、保護者の共感を引く。

### パターン4：体験エピソード型
```
先日、小学4年生の生徒からこんな質問を受けました。
```
→ 実際の現場エピソードを冒頭に配置して信頼感を確立する。

## 文体ルール

### トーン・スタイル
- **一人称**: 「私」
- **読者への呼びかけ**: 「保護者の方」「親御さん」「保護者・教育者の皆さん」
- **語尾**: 「です・ます」調。堅すぎず、親しみやすい語りかけ調。
- **強調**: `<strong>` タグを多用し、要点を太字で目立たせる。
- **絵文字**: 書き出しや見出しでアクセント的にのみ使用（🔵 🟢 🟡 💡 📊 📚 ✅）
- **方言**: 大分弁を話題にすることはあるが、記事本文は標準語。

### テキスト構成
- 1段落は2〜3文以内。読みやすく短くまとめる。
- 「」で読者の内心や疑問を代弁し、それに答える流れを多用。
- 比喩や具体例を積極使用：「ブレーキのない車に乗せるようなもの」など。

### HTMLタグの使用
- `<strong>` で重要フレーズを太字にする（Markdown の `**` ではなく Hugo の HTML 直接記述も多用）
- `<br>` で表内改行
- `<table>` でアフィリエイトリンク埋め込み
- `<div>` で強調ボックスを作成

### タイトル命名規則
- **形式**: `【修飾語】キーワード含むタイトル`
- **例**:
  - `【2025年版】小学生向けゲーミングPCおすすめ6選｜フォートナイト・マイクラ快適`
  - `【実践】バイブコーディングとは？Google AI Studioで中学生がお絵かきアプリを作ってみた！`
  - `【保護者必見】2026年、エンジニアの働き方が激変！`
  - `【AI教育】「その動画、本物？」と疑える子へ。`
- 【】の修飾語候補: `2025年版`, `実践`, `保護者必見`, `AI教育`, `体験レポート`, `10年後の未来`, `実録`, `自由研究`, `パパママ必読`

### 記事構成のフロー

```
1. 冒頭（リードセクション）
   - 書き出し（パターン1〜4のいずれか）
   - 読者の疑問・不安を「」で代弁
   - 記事の目的と読後の価値を宣言

2. 本文（H2×3〜6セクション）
   - 各 H2 に独自の視点・データ・体験を配置
   - 比較表（Markdownテーブル）を積極活用
   - blockquote で引用データや会話例を挿入
   - 具体例はナンバリング（1. 2. 3.）で整理

3. まとめセクション
   - H2: 「まとめ：〇〇」
   - 比較表で記事のエッセンスを凝縮
   - 保護者へのアクション提案（> blockquote 形式）

4. 参考資料
   - H2: 「参考リンク（References）」or「📚 参考資料・出典」
   - 公式1次情報のURL付きリスト
   - ※ 各データの発表時期を明記

5. 関連記事
   - H2: 「関連記事」
   - 内部リンク3〜4本（タイトル付きリスト）
```

## 執筆における4大原則

### 1. 「保護者目線」で書く
- 読者は「AI教育について詳しくない保護者」が中心。
- 専門用語は必ず噛み砕いて説明する。
- 「お子さん」「我が子」「親御さん」等の表現を自然に使う。

### 2. 「体験ファースト」で説得力を出す
- 自分の授業・開発体験を記事に散りばめる。
- 生徒との会話を `>` blockquote で引用する。
- 「私も最初は〇〇でした」と共感ポイントを入れる。

### 3. 「公式データ」で信頼性を担保する
- 世界経済フォーラム（WEF）、文部科学省、UNESCO、NRI 等の1次情報を必ず引用。
- 出典URLは記事末尾に参考資料リストとしてまとめる。
- 統計データは `>` blockquote + 📊 絵文字で視覚的に目立たせる。

### 4. 「家庭でできるアクション」で締める
- 記事の最後には必ず、保護者が「今日からできること」を提示する。
- `> **保護者の方へ**` ブロックで具体的なアクション・ツールリンクを提供。
- 「お子さんにこう聞いてみてください」型の締めが最も特徴的。

## SEO・メタデータ戦略

### description の書き方
- 120〜160文字。
- 必ずターゲットキーワードを含める。
- 読者のベネフィットを明示する：「〇〇を▲▲字で解説します」
- 例: `「日本のAI教育は遅れてる？」世界各国の最新事情を徹底リサーチ！アメリカのAIリテラシー、韓国のAIデジタル教科書、シンガポールの国家戦略など、親として知っておきたいグローバルスタンダードを3000字で解説します。`

### サムネイル
- ファイルパスは `img/` か `images/フォルダ名/` 配下。
- アニメ風のイラストが基本スタイル。

### 内部リンク戦略
- 記事末尾に「関連記事」セクションで3〜4本の内部リンク。
- 相互リンクを意識し、新しい記事から過去記事へ、過去記事から新記事へリンクを張る。

## 記事ジャンル別テンプレート

### 体験レポート記事
```
冒頭: 体験の経緯・きっかけ
  ↓
準備: 使ったツール・環境
  ↓
実践: ステップバイステップの実録（スクショ付き）
  ↓
結果: 完成物の紹介（機能リスト）
  ↓
学び: 教育的意義・子供の反応
  ↓
まとめ: 保護者へのアクション提案
```

### 比較・おすすめ記事
```
冒頭: 保護者の悩み・疑問を代弁
  ↓
選び方の基準: スペック表で整理
  ↓
おすすめN選: 各製品の詳細（価格・特徴・こんな子に）
  ↓
注意点: 親の心配事への対策
  ↓
まとめ: 比較表で総括
```

### ニュース・解説記事
```
冒頭: なぜ今この話題が重要か
  ↓
データ: 公式統計・調査結果をblockquoteで提示
  ↓
各国/各社の動き: H2で国・企業別に分割
  ↓
日本の家庭への影響: 保護者目線に翻訳
  ↓
まとめ: キーワード3つ程度でまとめ + 家庭でできること
  ↓
参考リンク: 1次情報URL一覧
```

### AIツール活用記事
```
冒頭: ツールが解決する課題を提示
  ↓
ツール概要: 他のAIとの違い・強み
  ↓
新機能紹介: 各機能をH2で個別解説（スクショ付き）
  ↓
実践テクニック: おすすめプロンプト・使い方のコツ
  ↓
実録: 講師自身の活用事例
  ↓
注意点: 保護者向けの留意事項
  ↓
まとめ: 比較表 + 「まずは〇〇してみてください」
```

## 注意事項

1. **過度な煽り禁止**: 「絶対」「確実」等の断言は避け、「可能性が高い」「と考えられます」を使用。
2. **AI盲信への警鐘**: AIの限界（ハルシネーション、著作権リスク）は必ず言及する。
3. **「人間中心」の原則**: AIは道具であり、人間が主体的に使いこなすべきという立場を一貫して保つ。
4. **年齢制限の明記**: GoogleのAIサービスの年齢制限について保護者に注意喚起する。
5. **アフィリエイト**: おすすめ記事では楽天/A8ネットのアフィリエイトリンクを表形式で埋め込み。
6. **コメント日本語**: コード内のコメントは日本語で記述する。
7. **エラーハンドリング**: API 呼び出しなどがある場合は必ずエラーハンドリングを行う。
